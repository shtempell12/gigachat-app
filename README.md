# GigaChat App

Веб-приложение в стиле ChatGPT, работающее на базе OpenAI API.

> После запуска добавьте скриншоты интерфейса в эту секцию.

## Функциональность

### Интерфейс чата
- Главный экран с областью сообщений и полем ввода
- Визуальное разделение сообщений пользователя и ассистента
- Markdown-форматирование ответов (заголовки, списки, таблицы, код, ссылки)
- Индикатор загрузки (typing indicator) с анимацией
- Автоматическая прокрутка к последнему сообщению
- Копирование ответа ассистента в буфер обмена
- Кнопка «Остановить генерацию»

### Управление чатами
- Боковая панель со списком всех чатов
- Создание нового чата с автоматическим названием на основе первого сообщения
- Переключение между чатами без потери данных
- Переименование чата (inline-редактирование)
- Удаление чата с подтверждением
- Поиск по названию и содержимому чатов
- Сохранение истории в `localStorage`

### Работа с GigaChat API
- SSE streaming — постепенное отображение ответа токен за токеном
- Fallback на обычный REST при недоступности streaming
- Контекст диалога (массив messages с ролями system/user/assistant)
- Настройка параметров: temperature, top_p, max_tokens, repetition_penalty
- Мультимодальный ввод — отправка изображений через GigaChat API

### Что покрыто

**Reducer (`src/store/chatReducer.test.ts`)**
- `CREATE_CHAT` — создаётся новый чат с уникальным id, устанавливается активным
- `ADD_MESSAGE` — сообщение добавляется в конец массива, другие чаты не затрагиваются
- `DELETE_CHAT` — чат удаляется, при удалении активного `activeChatId` сбрасывается или переключается
- `RENAME_CHAT` — название обновляется по id, другие чаты не затрагиваются

**localStorage (`src/utils/storage.test.ts`)**
- Данные сохраняются через `saveChats` и восстанавливаются через `loadChats`
- При невалидном JSON приложение не падает, возвращается пустой массив

**InputArea (`src/components/chat/InputArea.test.tsx`)**
- Кнопка заблокирована при пустом вводе, активна после набора текста
- Клик на кнопку и Enter вызывают отправку; сообщение из пробелов не отправляется
- После отправки поле очищается

**Message (`src/components/chat/Message.test.tsx`)**
- `role="user"` — выравнивание вправо, кнопка копирования отсутствует
- `role="assistant"` — выравнивание влево, кнопка копирования присутствует

**Sidebar (`src/components/Sidebar/Sidebar.test.tsx`)**
- При пустом поиске отображаются все чаты, при вводе список фильтруется в реальном времени
- Кнопка удаления показывает запрос на подтверждение; подтверждение удаляет чат, отмена — нет

## Установка и запуск

### Требования
- Node.js 18+
- npm или yarn
- Аккаунт на [Sber GigaChat Developer Portal](https://developers.sber.ru/studio/workspaces)

### 1. Клонируйте репозиторий

```bash
git clone <repo-url>
cd gigachat-app
```

### 2. Установите зависимости

```bash
npm install
```

### 3. Настройте переменные окружения

```bash
cp .env.example .env
```

Откройте `.env` и вставьте ключ:

```env
VITE_OPENAI_API_KEY=sk-...
```

> **Где получить ключ**: [platform.openai.com/api-keys](https://platform.openai.com/api-keys) → Create new secret key.

### 4. Запустите приложение

```bash
npm run dev
```

Откройте `http://localhost:5173` в браузере.

### 5. Сборка для продакшна

```bash
npm run build
npm run preview
```

## Примечание

OpenAI API поддерживает запросы напрямую из браузера — прокси не требуется.
